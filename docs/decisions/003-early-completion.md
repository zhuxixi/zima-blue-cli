# ADR 003: 支持提前完成机制

## 状态

✅ 已接受

## 背景

Agent 每轮固定 15 分钟会导致资源浪费：
- 任务可能在 5 分钟内完成
- 却还要等待 10 分钟才能进入下一轮

## 决策

Agent 支持**提前完成**，任务完成后立即进入休眠。

## 机制

```python
if result.status == "completed" and elapsed < cycle_interval:
    # 提前完成，立即休眠
    sleep_time = 0
    print(f"✅ 任务提前完成！")
else:
    # 正常休眠至下一周期
    sleep_time = cycle_interval - elapsed
```

## 好处

### 1. 提高效率

- 异步任务启动后无需等待
- 快速修复可以立即进入下一轮

### 2. 节省资源

- 减少不必要的等待时间
- 更快响应状态变化

### 3. 更好的用户体验

- Agent 感觉更"敏捷"
- 减少"干等"的感觉

## 实现要点

1. **状态检测**: kimi-cli 在结果中报告 `status: "completed"`
2. **时间检查**: 计算实际执行时间
3. **立即休眠**: 跳过剩余等待时间
4. **快速唤醒**: 下一轮立即开始

## 场景示例

```
第 2 轮: 触发全量测试
├── 启动测试进程（2 分钟）
├── 记录 PID
├── 状态: completed（异步已启动）
└── 立即休眠（无需等待 13 分钟）

第 3 轮: 检查测试状态（5 分钟后）
├── 测试还在运行
├── 记录状态
└── 立即休眠
```
